Trabajamos un poco más el join left, el join right, además con primary key de varias columnas y un foreign key.

Un club administra distintas actividades  deportivas. En una tabla llamada "deportistas" guardamos los datos personales
y en una tabla denominada "inscripciones" almacenamos la información necesaria para las 
inscripciones de los deportistas en los distintos deportes.

1- Eliminamos  las tablas si existen.

2- Creamos las tablas:
 create table deportistas(
  dni char(8) not null,
  nombre varchar(30),
  domicilio varchar(30),
  primary key(dni)
 );

 create table inscripciones(
 dni char(8) not null, 
  deporte varchar(15) not null,
  año year,
  matricula char(1), /*si se ha pagado ='s' sino 'n'*/
  primary key(documento,deporte,año),
  FOREIGN KEY (dni) REFERENCES deportistas(dni)
 );

5- Realizamos un "left join" de la tabla "deportistas" a "inscripciones" buscando coincidencia de "dni":
 select d.documento,d.nombre,i.deporte,i.año,i.matricula
  from deportistas as d
  left join inscriptos as i
  on d.documento=i.documento;
Notad que los deportistas que no están inscritosen ningún deporte tiene la fila como "null".

5- Realice un "right join" para obtener la misma salida anterior:
 select d.documento,d.nombre,i.deporte,i.año,i.matricula
  from inscripciones as i
  right join deportistas as s
  on s.documento=i.documento;

7- Realice un "right join" desde la tabla "deportistas" a "inscripciones" buscando coincidencia de documento:
 select d.nombre,i.documento,i.deporte,i.año,i.matricula
  from deportistas as d
  right join inscripciones as i
  on d.documento=i.documento;
Notamos que la persona con documento "26333444" no se encuentra en "socios", la columna "nombre" 
(correspondiente a la tabla "socios") contiene "null".
